# Style Rules

## Unordered Lists (Bullets)

* use `*` for first level
  + use `+` for second level (positive attribute)
  - use `-` for second level (negative attribute)

## Headings and Bold

Reference a heading using bold markup.
For example, we are in the **headings and bold** section.

## Decorations

Do not add extra characters as decoration not specified in this document.

## Code

### Fences Backticks

Use `code` backticks around
* project or repository names. examples: `tagd`, `TAGL`, `tagsh`, `httagd`
* code of any type
* filenames

### Snippets

Use triple backticks with a language specifier.

## Naming

Name by domain role, not temporary representation.

Do not pollute semantics with artifacts from context such as:

* storage types
* refactor scaffolding
* temporary migration names

Remove temporary scaffolding names before task closeout.
