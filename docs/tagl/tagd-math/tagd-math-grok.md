# tagd Mathematical Foundations

**Formal Description of the Mathematics in the tagd Semantic-Relational System and TAGL**

This document captures the mathematics already present in the tagd system. It builds strictly from the source code using only concepts observable in the implementation.

## 1. Introduction

A **tagspace** is a finite set of tags together with two kinds of relations:

- **Identity relations** (vertical): exactly one subordinate relation per tag, inducing a rooted tree via ranks.
- **Predicate relations** (horizontal): zero or more additional labeled relations attached to each tag.

All semantics derive from structure. The system is deterministic and inspectable because its canonical form (`dump()`) is a pure textual representation of this structure.

---

## 2. Primitives

### 2.1 Tag
A tag consists of:
- a globally unique UTF-8 label `_id` (nominal identity)
- exactly one subordinate relation (structural identity)

### 2.2 Subordinate Relation (Identity Relation)
A subordinate relation is the pair:
\[
(sub\_relator,\ super\_object)
\]
where both are tags.

**Invariant** (enforced by source): Every tag except the root `_entity` has exactly one such relation.

### 2.3 Rank
The `rank` of a tag is a sequence of UTF-8 code points.

**Core rule** (from `rank.h` and `rank::contains()`):
\[
A\ \_sub\ B \iff \operatorname{rank}(B)\ \text{is a prefix of}\ \operatorname{rank}(A)
\]

This defines **containership**: every descendant’s rank contains its ancestor’s rank as a prefix.

---

## 3. Core Model – The Tagspace as a Poset

A tagspace \( (T, \preceq) \) is a **partially ordered set (poset)** under the containership order:
\[
A \preceq B \iff \operatorname{rank}(B)\ \text{is a prefix of}\ \operatorname{rank}(A)
\]

- The order is reflexive, antisymmetric, and transitive.
- It forms a **rooted tree** with root `_entity` (the unique minimal element, rank \( \epsilon \)).

**Source reference**: `rank.h`, `abstract_tag::operator<`, and `merge_containing_tags()` in `tagd.cc`.

---

## 4. Topology Induced by the Poset (Alexandrov Topology)

The poset induces a natural topology called the **Alexandrov topology**:

- **Open sets** = upward-closed sets = subtrees (if a tag is included, all its descendants are included).
- **Closed sets** = downward-closed sets = paths from the root to a tag.

This matches the source exactly: queries and `rank::contains()` operate on subtrees. The topology encodes reachability and inheritance directly from the rank prefix order.

---

## 5. Hard Tags as Universal Base

Every tagspace shares the same fixed hard tag subtree (from `hard-tags.h` and bootstrap). This is the **invariant base**:

- `_entity` is the root.
- Hard tags (`_sub`, `_rel`, `_number`, `_interrogator`, etc.) form the common skeleton.
- Every user-defined tag extends this base.

Mathematically, the hard tag structure is a **fixed subspace** that every tagspace contains by construction. Any map between tagspaces must preserve this base.

---

## 6. Predicates – Horizontal Structure

Predicate relations (`_has`, `_can`, `_refers_to`, etc.) are additional labeled relations attached to tags. They do **not** affect the poset order or rank.

They sit on top of the topological skeleton as extra structure on the nodes.

---

## 7. Lexical Structure and Syntax

TAGL is the language for constructing and querying the poset + predicates:

- `>> subject sub_relator super_object` adds a new element to the poset.
- Predicates attach horizontal relations without changing position in the tree.

The grammar in `parser.y` enforces the single-identity invariant at parse time.

---

## 8. Query Semantics

Queries traverse the poset using the containership order:
- Subtree queries (`_sub`) return upward-closed sets.
- Wildcard `*` matches any relator within the subtree.

This is exactly the topological notion of open sets generated by the partial order.

---

## 9. Canonical Form and Serialization

The `dump()` method produces a total ordering of the poset that respects the rank prefix order, yielding a canonical textual representation of the entire structure.

---

## 10. Mathematical Summary

A tagspace is a structure \( (T, \preceq, P) \) where:

- \( (T, \preceq) \) is a finite rooted tree poset equipped with the Alexandrov topology induced by rank prefixes.
- \( P \) is a set of labeled predicate relations on \( T \).
- Every element has exactly one covering relation in \( \preceq \).
- The hard tag subtree is the universal invariant base shared by all tagspaces.

TAGL is the language for constructing and querying this structure.
